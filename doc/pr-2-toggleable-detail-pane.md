## Body (copy-paste)

```markdown
## Summary
- Adds a "Show Detail Pane" preference (checkbox, defaults to `true` so existing users see no change)
- Adds a runtime toggle action `Toggle Detail Pane` with shortcut ⌘⇧D so users can hide/show the pane without leaving the command

## Why
When the detail pane is visible, the left list column is narrow and longer task descriptions get truncated. Users who don't need the metadata fields (scheduled date, start date, recurrence, file path) want full-width descriptions. This adds an opt-out preference plus a quick runtime toggle for occasional access.

## Test plan
- [x] `npm run lint` passes
- [ ] First open: pane visible (preference default)
- [ ] Turn `Show detail pane` off in preferences → reopen → list is full-width
- [ ] Cmd+Shift+D toggles the pane within the command without restart
```
